#01
Moonshot 因算力紧缺暂停 Kimi K3 新订阅：开源万亿旗舰第一次“卖到缺货”
Moonshot / Kimi K3周日 07-19 起7天高频信号

Global Times、Securities Times 报道：Moonshot 于 7 月 16 日发布 2.8 万亿参数开源大模型 Kimi K3，因用户请求暴涨、算力紧缺，7 月 19 日起暂停新 C 端订阅。这是开源万亿参数旗舰首次因真实需求而出现产能瓶颈，也侧面印证“开源已能承接一线流量”。

💬 点评

“卖到缺货”比任何 benchmark 都更能说明问题：开源旗舰的真实需求已经超过了算力供给，这是市场投票。

🎯 业务视角

给客户做选型时，除了看分数，必须把“可获得性 / 交付确定性 / 算力供给”写进评估：优先能稳定拿到权重或稳定 API 的开源档，避免押注一个买不到算力的“最强模型”。

🔬 技术背景

2.8T MoE + 1M context + 原生视觉，推理侧对 KV cache、MoE 路由、长上下文显存与批处理调度要求极高，算力紧缺本质是服务侧的吞吐/显存瓶颈。

🔗 上下游影响

利好推理框架、量化、KV cache 共享、多云弹性调度与“模型可切换”的中间层；也倒逼企业做供应商冗余。

🚀 6个月产品卖点预测

未来 6 个月，“开源旗舰私有化交付包 + 弹性算力调度 + 供应商切换开关”会成为高价值增购项。
